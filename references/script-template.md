# 剧本写作模板（Stage 3 Script Template）

> Stage 3 剧本是整个 8 阶段流水线的核心枢纽。上游承接创意方向，下游支撑审查和资产拆解。
> **必须**同时满足「叙事质量」和「AI 可生产性」两个标准。

---

## 剧本标准结构

```markdown
# [项目名] — 剧本 v[X]

> **模式**: [15s emotion short | 30-60s theme short | 1-3min narrative short]
> **总时长**: XXs
> **创意方向**: [选自 Stage 2 的方向编号]
> **生成日期**: YYYY-MM-DD
> **状态**: [DRAFT / UNDER_REVIEW / APPROVED]

## 五、台词 / 旁白策略

### 台词设计原则
- **极简优先**: AI 短视频中台词越少越好，能用画面的不用台词
- **每句台词必须有画面功能**: 推进剧情 / 揭示性格 / 制造反差
- **英文原版 + 中文备注**: 最终输出用英文（Seedance 兼容），中文作注释

## 八、连续性备忘录（给制片主任的提示）

> 这部分是 Stage 5 资产拆解的关键输入。

### 角色连续性
- [ ] 每个出场角色的外貌必须在所有场次保持一致
- [ ] 服装状态变化（如有战斗损伤/环境痕迹）需标注时间点
- [ ] 道具状态变化（如武器消耗/损坏）需标注

### 空间连续性
- [ ] 各场景之间的空间关系是否合理
- [ ] 光源一致性（同一场景内光线不能突变）
- [ ] 天气/时间推移（如有）需有明确标记

## 九、审查预检（写完自查）

```markdown
**✅ 剧本预检清单**:
- [ ] 一句话命题清晰且可执行
- [ ] 情绪曲线有明确的起承转合
- [ ] 每个场景都有独特的叙事功能
- [ ] 角色行为符合已建立的性格
- [ ] 对白极简且每句都有功能
- [ ] 运镜建议覆盖了主要节拍（为 Stage 7 分镜预留接口）
- [ ] 所有视觉元素都可在 AI 生图/生视频工具中实现
- [ ] 总时长符合所选模式的约束
- [ ] 连续性备忘录已填写完整
- [ ] 风格锚定引用了具体作品/工作室
```
