# Seedance Package Spec

Use this file after assets are prepared and storyboard timing is known.

---

## 1. Goal

Generate a Seedance 2.0 package that is:
- Faithful to the approved script.
- Bound to approved assets and storyboard frames generated by `image2`.
- Programmed strictly around a **maximum of 15 seconds per segment**.
- Structured for copy-pasteable production, utilizing seedance2.0's **All-Round Reference (全能参考)** to lock characters, scenes, props, and pacing.

---

## 2. Package Structure

### 1. Master Control Section
Include:
- Project theme.
- Visual tone.
- Emotional tone.
- Overall motion and pacing tendency.

### 2. Asset & Storyboard Reference Table
Map:
- `@角色` (Characters)
- `@场景` (Scenes)
- `@道具` (Props)
- `@故事板` (Storyboard frames)

Each reference should point to the respective generated `image2` file.

### 3. Chronological Timeline Table
For each 15s segment block, include:
- Segment ID (e.g. `SEG01_0-15s`).
- Segment Time Range.
- Linked image2 Assets and Storyboard frame (`@故事板-01`).
- Action Chain & Camera Staging.
- Transition Safe Zones.

### 4. seedance2.0 Execution Prompts
Each segment prompt must adopt the standard playbook template and enforce the 8 rules in [prompt-compilation-playbook.md](prompt-compilation-playbook.md), using English for the prompt body to maximize model fidelity.

---

## 3. Strict 15-Second Segmenting & Spanning Rules

Since seedance2.0 cannot generate more than 15s at a time:
- **Strict 15s Limit**: Every segment ID must be partitioned into blocks of at most 15 seconds:
  - `SEG01_0-15s`
  - `SEG02_15-30s`
  - `SEG03_30-45s`
- **Spanning Scenes (跨段衔接)**: If a scene cannot be fully explained within 15 seconds, split it across subsequent segments (e.g. `SEG01_0-15s` and `SEG02_15-30s`).
  - In `SEG02_15-30s`, explicitly instruct seedance2.0 to load the ending frame of `SEG01` as a motion start, and bind the same `@角色-001` and `@场景-001` to maintain seamless visual continuity.
- **Reference Binding (全能参考)**: Every prompt block must include explicit instructions mapping the `image2` generated assets:
  - `Character Reference`: `@角色-001.png`
  - `Scene Reference`: `@场景-001.png`
  - `Storyboard Reference`: `@故事板-01.png`
- **Safe Zone Handles**: Leave 1.5 seconds at the head and tail of every 15s segment with stable micro-motions. This gives the compiler stable handle space to stitch the segments together without generating warp artifacts.

---

## 4. Constraint Rules

- **Bilingual Staging**: All assets and storyboards must have bilingual prompts (English + Chinese) inside the handover file so the user can easily generate them in `image2` first.
- **Playbook Compliance**: Every prompt must pass a sanity check against the 8 industrial prompting rules in [prompt-compilation-playbook.md](prompt-compilation-playbook.md).
- **All-Round Reference Execution**: Never write loose action prose without anchoring it to the generated assets and storyboard references.
